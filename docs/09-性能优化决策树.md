## 🖥️ CPU 优化决策树

### 决策流程

```mermaid
flowchart TD
    Start([CPU使用率过高]) --> Check1{top/htop观察}
    
    Check1 -->|用户态CPU高| User[用户态分析]
    Check1 -->|系统态CPU高| Sys[系统态分析]
    Check1 -->|IO等待高| IO[IO瓶颈分析]
    
    User --> User1[生成火焰图]
    User1 --> User2{热点代码类型?}
    
    User2 -->|业务计算| Biz[业务逻辑优化]
    User2 -->|集合操作| Collection[集合优化]
    User2 -->|字符串处理| String[字符串优化]
    User2 -->|序列化| Ser[序列化优化]
    
    Biz --> Biz1[算法优化]
    Biz --> Biz2[缓存结果]
    Biz --> Biz3[并行计算]
    
    Collection --> Col1[选择合适集合类型]
    Collection --> Col2[预估初始容量]
    Collection --> Col3[避免频繁扩容]
    
    String --> Str1[使用StringBuilder]
    String --> Str2[避免正则表达式]
    String --> Str3[使用字符串池]
    
    Sys --> Sys1[检查系统调用]
    Sys --> Sys2[减少线程切换]
    Sys --> Sys3[检查锁竞争]
    
    IO --> IO1[异步IO]
    IO --> IO2[批量读写]
    IO --> IO3[缓存优化]
    
    User2 -->|锁竞争| Lock[锁优化]
    Lock --> Lock1[缩小锁粒度]
    Lock --> Lock2[使用读写锁]
    Lock --> Lock3[无锁数据结构]
    Lock --> Lock4[减小临界区]
```

### 快速检查清单

| 检查项 | 命令/工具 | 正常值 | 异常处理 |
|--------|----------|--------|----------|
| CPU使用率 | `top` | < 70% | 分析火焰图 |
| 上下文切换 | `vmstat 1` | < 10万/秒 | 检查锁竞争 |
| 线程CPU分布 | `ps -mp <pid> -o THREAD,tid,time` | 均衡 | 检查线程亲和性 |

## 💾 内存优化决策树

### 决策流程

```mermaid
flowchart TD
    Start([内存使用异常]) --> Check1{jmap -heap <pid>}
    
    Check1 -->|堆内存高| Heap[堆内存分析]
    Check1 -->|堆外内存高| OffHeap[堆外内存分析]
    
    Heap --> Heap1{内存使用模式?}
    Heap1 -->|持续增长不释放| Leak[内存泄漏排查]
    Heap1 -->|周期性峰值| Peak[峰值优化]
    Heap1 -->|老年代快速增长| OldGen[对象生命周期优化]
    
    Leak --> Leak1[生成堆转储]
    Leak1 --> Leak2{泄漏源类型?}
    
    Leak2 -->|静态集合| Static[清理静态集合]
    Leak2 -->|ThreadLocal| TL[清理ThreadLocal]
    Leak2 -->|未关闭资源| Resource[try-with-resources]
    Leak2 -->|缓存无限增长| Cache[设置容量上限]
    Leak2 -->|监听器未注销| Listener[及时注销]
    
    Peak --> Peak1[对象池化]
    Peak --> Peak2[流式处理]
    Peak --> Peak3[分批处理]
    
    OldGen --> OG1[减少对象引用链]
    OldGen --> OG2[使用弱/软引用]
    OldGen --> OG3[及时置null]
    
    OffHeap --> OH1[检查DirectBuffer]
    OffHeap --> OH2[检查JNI内存]
    OffHeap --> OH3[检查NIO内存映射]
```

### 内存泄漏常见模式

```mermaid
flowchart LR
```

## 🗑️ GC 优化决策树

### 决策流程

```mermaid
flowchart TD
    Start([GC问题]) --> Check1{GC日志分析}
    
    Check1 --> Check2{GC问题类型?}
    
    Check2 -->|Young GC频繁| YGC[新生代优化]
    Check2 -->|Full GC频繁| FGC[Full GC优化]
    Check2 -->|GC停顿长| Pause[低延迟优化]
    Check2 -->|GC后内存不释放| Release[内存泄漏]
    
    YGC --> YGC1{对象分配特征?}
    YGC1 -->|短生命周期对象多| YGC2[增大Eden区]
    YGC1 -->|大对象多| YGC3[直接进入老年代]
    YGC1 -->|晋升快| YGC4[调整晋升阈值]
    
    FGC --> FGC1{触发原因?}
    FGC1 -->|老年代满| OldFull[老年代优化]
    FGC1 -->|元空间满| Meta[增大Metaspace]
    FGC1 -->|System.gc| SysGC[禁用显式GC]
    
    OldFull --> OF1[增大老年代]
    OldFull --> OF2[减少大对象]
    OldFull --> OF3[优化对象生命周期]
    OldFull --> OF4[检查内存泄漏]
    
    Pause --> Pause1{应用类型?}
    Pause1 -->|Web应用| Pause2[使用G1/ZGC]
    Pause1 -->|低延迟金融| Pause3[使用ZGC/Shenandoah]
    Pause1 -->|批处理| Pause4[使用Parallel GC]
    
    Pause2 --> P21[设置目标停顿时间]
    Pause2 --> P22[优化Region大小]
    
    Pause3 --> P31[ZGC配置]
    Pause3 --> P32[Shenandoah配置]
```

### GC 选择决策表

| 应用场景 | 推荐GC | 目标停顿 | 配置示例 |
|---------|--------|---------|---------|
| 低延迟 (< 10ms) | ZGC | < 1ms | `-XX:+UseZGC -XX:ZCollectionInterval=5` |
| 低延迟 (< 100ms) | G1 | < 200ms | `-XX:+UseG1GC -XX:MaxGCPauseMillis=200` |
| 高吞吐 | Parallel | 不敏感 | `-XX:+UseParallelGC` |
| 小内存 (< 100MB) | Serial | < 1s | `-XX:+UseSerialGC` |

## 📝 决策速查表

### 症状 → 工具 → 方案 映射

| 症状 | 诊断工具 | 关键指标 | 常用方案 |
|------|---------|---------|---------|
| CPU高 | async-profiler | 热点方法 | 缓存、算法优化 |
| 内存高 | jmap + MAT | 大对象、泄漏源 | 对象池、弱引用 |
| 响应慢 | Arthas trace | 耗时分布 | 异步、缓存、索引 |
| GC频繁 | GC日志 + gceasy | GC频率、停顿 | 调堆大小、换GC |
